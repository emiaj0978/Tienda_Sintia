# Tienda Sintia

---

## 1. Descripción del negocio

La **Tienda Sintia** es un negocio dedicado a la venta de productos de primera necesidad como alimentos, bebidas, productos de limpieza y artículos básicos del hogar.

Su objetivo es brindar atención rápida y eficiente a los clientes del barrio, ofreciendo productos accesibles y de consumo diario.

---

## 2. Identificar el problema y solución

### Problema

Actualmente la bodega presenta los siguientes problemas:

- No existe un control adecuado del stock.
- Las ventas se registran manualmente o no se registran.
- Se generan pérdidas de productos.
- No se sabe cuáles son los productos más vendidos.
- Hay desorden en la información.

---

### Solución

Se propone desarrollar un **sistema de gestión** que permita:

- Controlar el inventario en tiempo real.
- Registrar automáticamente las ventas.
- Gestionar productos de forma organizada.
- Obtener reportes de ventas.

---

## 3. Preanálisis

### Necesidades

- Registro de productos.
- Control de inventario.
- Registro de ventas.
- Control de ingresos.
- Reportes básicos.

---

### Estudio de viabilidad

- **Técnica:** Desarrollo posible con Java y MySQL.
- **Económica:** Bajo costo (solo desarrollo del sistema).
- **Operativa:** Fácil de usar por el dueño o trabajador.

---

### Alcance del sistema

El sistema permitirá:

- Gestionar productos.
- Registrar ventas.
- Controlar stock.
- Generar reportes.

**No incluye:**
- Ventas online.
- Delivery.

---

## 4. Análisis

### Definición de requisitos

#### Requisitos funcionales

- Registrar productos.
- Editar productos.
- Eliminar productos.
- Registrar ventas.
- Consultar stock.
- Generar reportes de ventas.

---

#### Requisitos no funcionales

- Interfaz sencilla.
- Sistema rápido.
- Seguridad de datos.
- Funcionamiento sin internet.

---

### Análisis de requisitos

El sistema debe:

- Ser intuitivo.
- Evitar errores en ventas.
- Mostrar información clara.
- Permitir futuras mejoras.
